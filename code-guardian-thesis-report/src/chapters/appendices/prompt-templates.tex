\chapter{Prompt Templates}
\label{app:prompt-templates}

This appendix records the prompt structures used in the evaluated prototype. Prompt wording and schema constraints materially affect quality, parse robustness, and false-positive behavior (see Section~\ref{sec:eval-rag-sensitivity}).

\section{LLM-Only Detection Prompt Template}
\label{sec:prompt-llm-only}

\subsection{System Prompt}
\begin{lstlisting}[basicstyle=\small\ttfamily, breaklines=true, frame=single, caption={LLM-only system prompt}, label={lst:system-prompt}]
You are a security analysis assistant.
Analyze the code snippet for security vulnerabilities.

Return ONLY valid JSON following the required schema.
If no vulnerability is found, return {"issues": []}.

For each issue include:
- vulnerability type
- CWE (if known)
- severity
- line number(s)
- short explanation
- optional code-level fix
\end{lstlisting}

\subsection{JSON Schema}
\begin{lstlisting}[language=json, basicstyle=\small\ttfamily, breaklines=true, frame=single, caption={Expected JSON output schema}, label={lst:json-schema}]
{
  "issues": [
    {
      "type": "string",
      "cwe": "string",
      "severity": "high|medium|low",
      "line": "number",
      "description": "string",
      "fix": "string (optional)"
    }
  ]
}
\end{lstlisting}

\subsection{Code Context Injection}
\begin{lstlisting}[basicstyle=\small\ttfamily, breaklines=true, frame=single, caption={Code context injection}, label={lst:code-context}]
Analyze the following code:

```javascript
<CODE_SNIPPET_HERE>
```

Respond with JSON only.
\end{lstlisting}

\subsection{Composed LLM-Only Prompt}
\begin{lstlisting}[basicstyle=\footnotesize\ttfamily, breaklines=true, frame=single, caption={Composed LLM-only prompt}, label={lst:complete-llm-prompt}]
<SYSTEM_PROMPT>
<JSON_SCHEMA>
<CODE_CONTEXT_BLOCK>
\end{lstlisting}

\textbf{Approximate token range:} 320--400 (varies by snippet length).

\section{LLM+RAG Detection Prompt Template}
\label{sec:prompt-rag}

RAG mode injects retrieved security guidance before the code snippet.

\subsection{Retrieval Injection Block}
\begin{lstlisting}[basicstyle=\small\ttfamily, breaklines=true, frame=single, caption={RAG knowledge injection}, label={lst:rag-injection}]
## Relevant Security Knowledge
<TOP_K_RETRIEVED_CHUNKS>

Now analyze the code with this knowledge in mind.

Analyze the following code:
```javascript
<CODE_SNIPPET_HERE>
```

Respond with JSON only.
\end{lstlisting}

\subsection{Retrieval Parameters}
\begin{itemize}
  \item \textbf{Embedding model:} \texttt{all-MiniLM-L6-v2}
  \item \textbf{Vector store:} local Chroma DB
  \item \textbf{Top-k:} 5 retrieved chunks
  \item \textbf{Similarity metric:} cosine similarity
  \item \textbf{Chunk length:} roughly 200--500 tokens
\end{itemize}

\subsection{Composed LLM+RAG Prompt}
\begin{lstlisting}[basicstyle=\footnotesize\ttfamily, breaklines=true, frame=single, caption={Composed LLM+RAG prompt}, label={lst:complete-rag-prompt}]
<SYSTEM_PROMPT>
<JSON_SCHEMA>
<RAG_KNOWLEDGE_BLOCK>
<CODE_CONTEXT_BLOCK>
\end{lstlisting}

\textbf{Approximate token range:} 850--1200 (about 2.5--3$\times$ LLM-only).

Task: Fix the vulnerability while preserving intended behavior.

Vulnerability details:

\section{Prompt Engineering Notes}

\paragraph{JSON contract enforcement.}
Strict ``JSON only'' instructions and defensive parsing were necessary to keep parse success near 100\%.

\paragraph{Abstention remains under-specified.}
The evaluated prompts allow empty findings but do not strongly enforce abstention on secure code; this contributes to elevated secure-sample FPR in some configurations.

\paragraph{RAG ordering matters.}
Retrieved knowledge is placed before code so security guidance is in context before generation.

\section{Reproducibility Notes}

To reproduce the reported behavior:
\begin{enumerate}
  \item Use the same prompt structure as Listings~\ref{lst:complete-llm-prompt} and~\ref{lst:complete-rag-prompt}.
  \item Keep generation settings fixed (\texttt{temperature=0.1}, \texttt{num\_predict=1000}).
  \item Use the same retrieval setup (\texttt{k=5}, cosine similarity, \texttt{all-MiniLM-L6-v2}).
  \item Use the same model artifacts and digests reported in the evaluation chapter.
\end{enumerate}

Small wording changes can shift model behavior; this appendix reflects the final evaluated template family.
