\label{sec:r1-consistency}

R1 addresses the reliability of the assistant's detection behavior. For Code Guardian, detection accuracy and consistency mean that the same code should lead to comparable findings, classifications, and localizations under fixed settings. This is especially important in security analysis, where unstable results quickly reduce developer trust and make remediation decisions harder to justify.

This requirement is motivated by two risks. First, traditional security tools already suffer from noisy warnings, and inconsistent output further lowers acceptance in practice \cite{johnson2013don}. Second, LLM-based systems introduce additional nondeterminism through probabilistic generation and schema deviations \cite{openai2023gpt4,ji2023hallucination,pearce2022copilot}. A practical IDE assistant must therefore control variation rather than merely maximize raw recall.

For this thesis, R1 implies stable issue detection, stable CWE-style classification, and sufficiently repeatable localization for developer action. The design response is a strict JSON contract, defensive parsing, scoped prompting, and optional retrieval grounding to reduce unsupported variation.

In the evaluation, R1 is operationalized primarily through pooled issue-level precision, recall, and F1 under fixed configurations, complemented by parse stability and qualitative checks of label and localization stability across repeated runs.
