This chapter evaluates Code Guardian against the requirements from Chapter~\ref{chap:analysis}. The focus is on three dimensions: detection quality (precision/recall/F1 and secure-sample FPR), structured-output robustness (JSON parse success), and responsiveness (latency in IDE-like workflows). Results are interpreted as descriptive measurements for the documented local run configuration.

\paragraph{Key observations.}
\begin{itemize}
  \item \textbf{Best overall quality in this run:} \texttt{qwen3:8b} with RAG (highest F1, moderate secure-sample FPR).
  \item \textbf{RAG is model-dependent:} helpful for \texttt{qwen3} variants, not universally beneficial across all models.
  \item \textbf{False-positive control is the main bottleneck:} most LLM configurations still over-warn on secure samples.
  \item \textbf{SAST remains a useful complement:} lower recall but much lower alert noise and latency.
\end{itemize}
