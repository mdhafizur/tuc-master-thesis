Secure coding support within the IDE is increasingly demanded, as developers prefer immediate feedback during implementation rather than deferred security checks. Traditional static analysis tools remain effective for rule-based vulnerabilities but struggle with weaknesses requiring semantic reasoning or domain knowledge. Large Language Models (LLMs) offer potential for vulnerability detection and repair; however, adoption is constrained by inconsistent quality, high false-positive rates, and privacy concerns when code is processed by cloud services.

This thesis investigates privacy-preserving vulnerability detection and repair by integrating locally deployed LLMs with Retrieval-Augmented Generation (RAG) in a VS Code extension. The system operates on-device, preserving source-code privacy while using locally cached CWE, CVE, and OWASP knowledge. Two modes are implemented: a low-latency inline mode for real-time feedback, and an audit mode combining LLM reasoning with retrieval-based context enrichment.

The system is evaluated on 128 JavaScript/TypeScript cases (113 vulnerable, 15 secure) aligned to CWE/OWASP concepts. Metrics include precision, recall, F1-score, false-positive rate, and latency. Results show model-dependent trade-offs: the best F1 (63.76\%) is achieved by \texttt{qwen3:8b} with RAG. The study demonstrates that privacy-preserving local LLM workflows provide useful vulnerability analysis, while model selection and false-positive control remain critical for practical deployment.
